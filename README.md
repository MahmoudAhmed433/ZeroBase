# ZeroBase

ZeroBase is a full-stack platform where students discover practical training posts from real companies.  
The UI uses a professional wooden pin-board style with paper-note cards.

## Clean Project Structure

```text
ZeroBase/
├── backend/                 # Django + DRF API
│   ├── backend/             # Project settings/urls
│   ├── core/                # Domain app (users, companies, posts, applications)
│   ├── manage.py
│   └── requirements.txt
├── frontend/                # Next.js app (App Router + Tailwind)
│   ├── app/
│   ├── components/
│   ├── lib/
│   └── package.json
├── .gitignore
└── README.md
```

Legacy static files were removed to keep only the production-ready stack.

## Stack

- Backend: Django REST Framework + JWT (`simplejwt`)
- Frontend: Next.js + TailwindCSS + Framer Motion
- Docs: drf-spectacular (Swagger + Redoc)

## Public-First Flow

- Home page is public and opens directly without login.
- Guests can browse/search posts.
- Authentication is optional.
- Course details page: `/courses/{id}` with full company info, rating, and testimonials.
- Protected pages:
  - `My Courses` (student)
  - `Company Dashboard` (company)
- Navbar includes clear `Sign In` and `Sign Up` actions.
- Sign-up starts with role question: **"Are you a company?"** then shows matching form.
- After successful Sign Up/Login, user is redirected to Home.
- Logged-in users see a single account dropdown menu (no duplicate Sign In/Sign Up buttons).

## Backend Run

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py seed_demo
python manage.py runserver
```

Backend URL: `http://localhost:8000`

## Frontend Run

```bash
cd frontend
npm install
```

Create `frontend/.env.local`:

```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api
```

Run:

```bash
npm run dev
```

Frontend URL: `http://localhost:3001`

## Quick Run Scripts (Makefile)

From project root:

```bash
make backend-setup
make backend-seed
make backend-run
```

In another terminal:

```bash
make frontend-setup
make frontend-run
```

If migrations are broken locally (custom-user migration history conflict), run:

```bash
make backend-reset
make backend-run
```

`backend-reset` recreates SQLite DB then runs migrate + seed demo data.

## Key Endpoints

- Auth:
  - `POST /api/auth/signup/`
  - `POST /api/auth/login/`
  - `POST /api/auth/refresh/`
  - `GET /api/auth/me/`
- Public data:
  - `GET /api/posts/`
  - `GET /api/companies/`
- Student:
  - `POST /api/applications/`
  - `GET /api/applications/my-courses/`
  - `GET /api/bookmarks/`
  - `POST /api/bookmarks/`
  - `DELETE /api/bookmarks/{id}/`
- Company:
  - `POST /api/posts/`
  - `GET /api/posts/company-dashboard/`
  - `GET /api/applications/company-candidates/`
  - `POST /api/applications/{id}/update-status/`

## Troubleshooting

- If homepage shows no cards, verify backend is running and `seed_demo` was executed.
- If CORS errors appear, ensure frontend uses `http://localhost:3001` and backend allows it.
- If login succeeds but role behavior is wrong, clear browser local storage and login again.
